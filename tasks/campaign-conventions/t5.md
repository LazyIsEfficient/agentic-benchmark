# Add revision history to notes

Continuing in `notes-store`. A `Note` (`src/note.ts`) now has a `title`, a
`body`, a `createdAt`, and `attachments`.

## Task

Track the history of body edits.

- Define a `Revision` type with three fields: a unique `id` (string), the `body`
  as it reads after the edit, and `at` — when the edit happened.
- Give `Note` a `revisions: Revision[]` field that starts empty.
- Add a method `update(title: string, body: string): boolean` to `NoteStore`.
  It finds the note by `title`, replaces its `body` with the new one, and
  appends a new `Revision` capturing the new body, a freshly generated unique
  id, and the time of the edit. It returns `true` on success, or `false` if no
  note matches `title` (changing nothing).

Add a unit test covering: an update appends exactly one revision (with an id and
an edit time), the note's `body` reflects the latest update, and updating a
missing note returns `false`.
