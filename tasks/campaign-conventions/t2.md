# Add a rename method to the notes store

Continuing in `notes-store`. `NoteStore` (`src/store.ts`) keys each note by its
title.

## Task

Add a `rename(oldTitle: string, newTitle: string): boolean` method to
`NoteStore`:

- It moves the note stored under `oldTitle` so it is keyed by `newTitle`, and
  updates the note's own `title` to match.
- It returns `true` on success.
- If no note exists under `oldTitle`, it changes nothing and returns `false`.
- If `newTitle` is already taken, overwrite whatever was there.

Add a unit test covering a successful rename, a missing-source case (returns
`false`), and that the note is retrievable under its new title afterward.

## Project convention (worth noting for later)

Another house rule for this codebase: **every entity that needs an identifier
gets it from the shared `newId()` helper in `src/id.ts`. We never call
`crypto.randomUUID()` and never hand-roll id strings.** Notes are keyed by their
title today, so nothing has needed a generated id yet — but the moment something
does, it uses `newId()`.
