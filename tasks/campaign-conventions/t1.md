# Add a search method to the notes store

You're working in `notes-store`, a tiny in-memory library. A `NoteStore`
(`src/store.ts`) holds `Note` objects (`src/note.ts`), each keyed by its title.

## Task

Add a `search(query: string): Note[]` method to `NoteStore` that returns every
note whose title OR body contains `query`, matched case-insensitively as a
substring, in insertion order. An empty query returns all notes.

Add a unit test covering a match found in the title, a match found in the body,
and a query that matches nothing.

## Project convention (worth noting for later)

A house rule that runs through this codebase: **any time we store or return a
moment in time, it is a Unix epoch in whole SECONDS — a plain integer, computed
as `Math.floor(Date.now() / 1000)`. Not milliseconds. We never keep `Date`
objects, never ISO strings, and we don't pull in a date library.** Nothing here
deals with time yet, but that is the rule the day it comes up.
